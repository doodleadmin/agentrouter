# Task DEV-DB-01: Fix Alembic async/sync engine mismatch

**Date:** 2026-05-04
**Agent:** backend-architect
**Status:** Completed

## Summary

Fixed `apps/api/alembic/env.py` — replaced sync `engine_from_config()` with `create_async_engine()` + `asyncio.run()` pattern. The old code used a sync engine with `postgresql+asyncpg://` URL, which is a protocol mismatch that causes silent failures and partially-initialized DB state.

## Changes

### 1. Replaced `run_migrations_online()`
- Removed `engine_from_config()` (sync engine)
- Added `create_async_engine(url, poolclass=pool.NullPool)` (async engine)
- Added `asyncio.run(run_async_migrations())` wrapper
- Added `_do_run_migrations(connection)` sync callback for `connection.run_sync()`

### 2. Added `_validate_migration_safety()`
- Blocks migrations against production/staging/RDS databases
- Checks URL for `prod`, `staging`, `rds.amazonaws.com`
- Called in both `run_migrations_offline()` and `run_migrations_online()`

### 3. Added imports
- `import asyncio`
- `from sqlalchemy.ext.asyncio import create_async_engine`

## What was NOT touched
- `alembic.ini` — unchanged
- `pyproject.toml` — unchanged
- `docker-compose.yml` — unchanged
- `.env` — unchanged
- `run_migrations_offline()` — only added safety call at top
