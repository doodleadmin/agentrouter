# 0001: Использовать Monorepo

Дата: 2026-05-03
Статус: accepted
Автор: studio-orchestrator

---

## Контекст
Нужно определить организацию репозитория для системы Agent Mission Control, которая состоит из нескольких сервисов: Orchestrator API, Telegram Bot Gateway, Celery Worker, React Dashboard, Memory vault, Infrastructure configs.

## Решение
Все сервисы разместить в одном git-репозитории (monorepo) с разделением на директории `apps/`, `infra/`, `docs/`, и `.ai_memory/` как главный Obsidian vault.

## Альтернативы

### Вариант A: Polyrepo
- Плюсы: Полная изоляция, независимые релизы
- Минусы: Дублирование моделей, сложнее координация

### Вариант B: Monorepo
- Плюсы: Общие модели, единый CI/CD, проще рефакторинг
- Минусы: Репозиторий больше, нужна дисциплина границ

## Последствия

### Положительные
- Общие SQLAlchemy-модели
- Единый CI/CD pipeline
- Memory vault рядом с кодом

### Отрицательные
- Репозиторий растёт со временем
- Нужна дисциплина границ модулей

## Затронутые компоненты
- Вся структура проекта
- CI/CD pipeline
- Деплой-процесс
