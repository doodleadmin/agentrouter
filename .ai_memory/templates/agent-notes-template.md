# Agent Notes Template

> Используется агентами для записи результатов задач в project memory.

---

## Формат записи

Каждый агент после завершения задачи добавляет секцию в `agent_notes.md` проекта:

```markdown
## YYYY-MM-DD / agent-slug / task-external-id

### Что сделано
- Краткий список изменений.

### Важно для будущих задач
- Ключевые находки.
- Нестандартные решения.
- Предупреждения.

### Изменённые файлы
- `path/to/file.py` — описание

### Следующие шаги
- Что нужно сделать дальше.
```

---

## Пример

```markdown
## 2026-05-03 / backend-architect / task-0001

### Что сделано
- Добавлен healthcheck endpoint GET /health.
- Добавлен тест test_health_ok.

### Важно для будущих задач
- Auth middleware не применяется к /health.
- Тесты запускаются через pytest.
- Используем httpx AsyncClient для тестов.

### Изменённые файлы
- `app/routers/health.py` — новый endpoint
- `tests/test_health.py` — новый тест

### Следующие шаги
- Добавить /version endpoint.
- Подключить Sentry.
```

---

## Правила

1. **Краткость** — не более 20 строк на задачу.
2. **Сигнал, не шум** — писать только то, что сэкономит время другим агентам.
3. **Без секретов** — никаких tokens, passwords, API keys.
4. **Дата обязательна** — формат YYYY-MM-DD.
5. **Agent slug обязательен** — кто писал.
6. **Task ID обязателен** — ссылка на задачу.
7. **Append-only** — не удалять чужие заметки.
