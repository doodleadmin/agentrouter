# Current State Template

> Шаблон для файла `current_state.md` на уровне проекта или глобально.

---

## Формат

```markdown
# [NAME] — Текущий статус

Обновлено: YYYY-MM-DD | Автор: agent-slug

---

## Статус проекта

**Фаза:** [фаза]
**Состояние:** [краткое описание]
**Блокеры:** [нет / описание]
**Критические проблемы:** [нет / описание]

## Что происходит сейчас

- [Активные работы]

## Активные задачи

| Задача | Статус | Агент |
|--------|--------|-------|
| ID: Название | ✅/🔄/❌ | agent-slug |

## Следующие шаги

1. [Следующие действия]

## Кодовая база

| Компонент | Статус | Файлы |
|-----------|--------|-------|
| Название | ✅ | путь |

## Memory vault статус

| Компонент | Статус |
|-----------|--------|
| Название | ✅/🔄 |
```

---

## Правила обновления

1. **Обновлять после каждой задачи** — агент обязан обновить current_state.md.
2. **Дата обязательна** — формат YYYY-MM-DD.
3. **Автор обязательна** — agent slug.
4. **Фаза из roadmap** — использовать фазы из docs/roadmap.md.
5. **Не удалять историю** — можно архивировать старые секции вниз.
6. **Без секретов** — никаких credentials.
